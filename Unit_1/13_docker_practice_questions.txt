7. Task 1 – Resource Mismanagement in Containers
Q1. Which container feature was misconfigured or missing?

👉 Answer: cgroups (memory limits not set)

docker run -d --name app_container nginx

❌ Above command has no memory limit, so container can consume unlimited RAM.

Q2. Which kernel mechanism should have prevented this?

👉 Answer: cgroups (memory control)

docker run -d \
  --memory="512m" \
  --memory-swap="512m" \
  --name app_container \
  nginx

✔ This ensures:

Max memory = 512MB
Container cannot exceed limit
Q3. Would namespaces alone solve this issue? Why?

👉 Answer: NO

# Namespaces provide isolation only (example)
docker run -d --name isolated_container nginx

❌ Namespaces only:

Isolate processes
Do NOT limit resource usage

✔ Resource control requires cgroups

Final Summary Commands
# Wrong (no limits)
docker run -d nginx

# Correct (with cgroups limits)
docker run -d --memory="512m" nginx
Conclusion
Missing feature → cgroups configuration
Required mechanism → cgroups (memory limits)
Namespaces alone → Not sufficient (only isolation, no control)

8. Task 2 – Container Resource & Isolation
Q1. Which Linux feature prevents Container A from consuming all CPU?

👉 Answer: cgroups (CPU control)

docker run -d \
  --cpus="1.0" \
  --name containerA \
  ubuntu

✔ Limits CPU usage so one container cannot consume all CPU resources

Q2. Which feature ensures Container B cannot see processes of Container C?

👉 Answer: PID Namespace

docker run -d \
  --name containerB \
  ubuntu

✔ Each container gets its own process space
✔ Processes are isolated and not visible across containers

Q3. Which namespace allows all three containers to use port 80 internally?

👉 Answer: Network Namespace

# Container A
docker run -d -p 8081:80 nginx

# Container B
docker run -d -p 8082:80 nginx

# Container C
docker run -d -p 8083:80 nginx

✔ Internally all containers use port 80
✔ Each container has its own network stack and IP

Final Summary
# CPU control → cgroups
docker run -d --cpus="1.0" ubuntu

# Process isolation → PID namespace
docker run -d ubuntu

# Network isolation → Network namespace
docker run -d -p 8080:80 nginx
Conclusion
CPU control → cgroups
Process isolation → PID namespace
Same port usage → Network namespace
